## Original Requirements

ATM Withdrawal Safety Backend

A transactional backend that processes ATM cash withdrawal requests
securely, with a minimal administrative console. It enforces balance
and limit rules, flags suspicious activity, and guarantees that any
failed transaction is fully rolled back.

Core features:
- Authenticate users by card number and PIN before any transaction, locking the account after 3 consecutive failed attempts
- Decline withdrawals exceeding the account balance or the daily limit per calendar day
- Process each withdrawal atomically, rolling back all steps completely if any step fails so the balance is unchanged
- Detect suspicious withdrawal patterns and flag the transactions for administrator review
- Let administrators review flagged transactions, lock or unlock accounts, and view transaction histories
- Record every authentication attempt and transaction state change with a timestamp

- As an administrator, I want to view complete historical logs of authentication attempts and transaction changes for auditing

## Requirement Analysis

The system requires a layered security approach. First, user authentication via card number and PIN with an account lockout mechanism after 3 failed attempts. Second, business rule validation ensuring withdrawals stay within balance and daily limits. Third, atomic transaction processing guaranteeing that if any step fails (insufficient funds, system error, etc.), the entire operation is rolled back entirely with no partial state changes. Fourth, a fraud detection engine that analyzes withdrawal patterns (e.g., rapid successive withdrawals, unusual amounts) and flags suspicious transactions. Fifth, an administrative console providing visibility into flagged transactions, account lock/unlock capabilities, and full transaction history. Finally, comprehensive audit logging for every authentication attempt and transaction state change, all timestamped. The backend must be stateless and built with RESTful principles for integration with ATM terminals.

## UI Design draft

The administrative console will feature a clean, minimal dashboard. Left navigation pane with sections: Dashboard (overview stats), Flagged Transactions (list with severity indicators), Account Management (search/lock/unlock), and Audit Logs (filterable by date, type, and account). Main content area uses data tables with sortable columns. Simple card-based layout for account details. Color coding: red for locked/flagged, green for active accounts. Timestamps displayed in ISO 8601 format throughout. Responsive design for desktop and tablet views.
