## Implementation approach

We will build a Python-based web application using Django as the core framework for its robust ORM, built-in authentication, and admin interface. For the gamified student interface, we'll use Django Channels for real-time features (live timer, animated nugget counter) and integrate with a lightweight JavaScript framework (Alpine.js) for reactive UI components. The competency heatmaps and radar charts will be generated using Plotly for interactive visualizations. The audit log will be implemented as an immutable database table with append-only triggers. For code snippet questions, we'll use a sandboxed execution environment (Pyodide or a simple Docker container) to run student code safely. The streak system and nugget wallet will be handled server-side with atomic database transactions to prevent race conditions. PostgreSQL will be the primary database for its support of JSON fields (for question types) and robust transaction isolation.

## Data structures and interfaces


classDiagram
    class User {
        -int id
        -str email
        -str password_hash
        -str role  # Admin, Instructor, Student
        -datetime created_at

        +has_permission(permission: str) -> bool
    }
    class StudentProfile {
        -int id
        -User user
        -str avatar_url
        -int nugget_balance
        -int current_streak
        -int max_streak
        +add_nuggets(amount: int, reason: str) -> None
        +deduct_nuggets(amount: int) -> bool
        +update_streak(correct: bool) -> None
    }
    class InstructorProfile {
        -int id
        -User user
        -str department
        +create_exam(exam_data: dict) -> Exam
        +view_analytics(exam_id: int) -> dict
    }
    class Exam {
        -int id
        -InstructorProfile creator
        -str title
        -str description
        -datetime open_date
        -datetime close_date
        -int time_limit_minutes  # per attempt
        -list[Cohort] cohorts
        +add_question(question: Question) -> None
        +get_competency_coverage() -> dict
        +is_active() -> bool
    }
    class Question {
        -int id
        -Exam exam
        -str question_type  # MCQ, DRAG_DROP, CODE
        -JSON content  # type-specific data
        -list[Competency] competencies
        -str difficulty  # BEGINNER, INTERMEDIATE, ADVANCED
        -float nugget_multiplier
        +get_correct_answer() -> str
        +evaluate_response(response: str) -> bool
    }
    class Competency {
        -int id
        -str name
        -str description
        -int mastery_threshold  # percentage
    }
    class Cohort {
        -int id
        -InstructorProfile creator
        -str name
        -list[StudentProfile] students
        +enroll_student(student: StudentProfile) -> None
        +get_leaderboard(window_days: int) -> list
    }
    class ExamAttempt {
        -int id
        -StudentProfile student
        -Exam exam
        -datetime start_time
        -datetime end_time
        -int total_nuggets_earned
        -int streak_multiplier
        -JSON responses  # question_id -> response
        -JSON results  # question_id -> {correct, nuggets, time_spent}
        +submit() -> dict  # returns competency breakdown
        +calculate_nuggets() -> int
    }
    class AuditLog {
        -int id
        -User actor

        -str resource_type

## Program call flow


sequenceDiagram
    participant Student as Student (Browser)
    participant UI as Student Interface (Alpine.js)
    participant WS as WebSocket Consumer
    participant View as Django View
    participant DB as PostgreSQL
    participant Audit as Audit Middleware

    Note over Student, Audit: Exam Taking Flow
    Student->>UI: Click "Start Exam"
    UI->>View: POST /exams/{id}/start
    View->>DB: Create ExamAttempt record
    View->>DB: Fetch first question
    View-->>UI: Return question data + timer info
    UI->>WS: Connect WebSocket for live updates
    WS->>WS: Start server-side timer countdown
    loop For each question
        UI->>Student: Display question (MCQ/DragDrop/Code)
        Student->>UI: Submit answer
        UI->>View: POST /exams/{id}/submit-answer
        View->>DB: Evaluate answer, update streak
        View->>DB: Calculate nuggets with streak multiplier
        View-->>UI: Return result (correct/incorrect, nuggets earned)
        UI->>WS: Send streak update
        WS-->>UI: Broadcast animated nugget counter
        alt Answer correct
            UI->>UI: Show green feedback, increment streak
        else Answer wrong
            UI->>UI: Show red feedback, reset streak to 0
        end
        UI->>View: GET /exams/{id}/next-question
        View-->>UI: Return next question or completion signal
    end
    Note over Student, Audit: Exam Submission
    UI->>View: POST /exams/{id}/submit
    View->>DB: Calculate final competency breakdown
    View->>DB: Update nugget balance, check badges
    View->>Audit: Log exam submission event
    View-->>UI: Return competency radar chart data + study tips
    UI->>Student: Display results with animations

    Note over Student, Audit: Instructor Analytics Flow
    Instructor->>UI: Navigate to exam analytics
    UI->>View: GET /analytics/exam/{id}
    View->>DB: Aggregate cohort performance
    View->>DB: Calculate per-competency heatmap
    View-->>UI: Return heatmap data + mastery thresholds
    UI->>UI: Render Plotly heatmap (red for below threshold)
