## Original Requirements

A medical inventory system that tracks blood unit stock in a hospital
blood bank. It manages compatibility between blood types, monitors
expiration, and raises alerts during shortages to support safe and
timely transfusions.

Core features:
- Track blood units by ABO type, Rh factor, and collection date, with expiry fixed at 42 days after collection
- Accept transfusion requests specifying blood type, Rh factor, and quantity
- Match requests to compatible units using standard ABO/Rh donor compatibility (exact-type matches first, then closest expiry first)
- Reserve matched units, and release a reservation back to stock if it is not confirmed as issued within 24 hours
- Automatically expire units past their expiry date and exclude them from matching
- Raise a shortage alert whenever available stock of any blood type falls below 5 units
- Show a dashboard of stock per type, units expiring within 7 days, and open requests

## Project Name

blood_bank_inventory_manager

## Product Goals

- Ensure accurate and efficient blood inventory tracking to minimize wastage
- Enhance transfusion safety through robust compatibility matching and expiry management
- Provide real-time visibility and proactive alerts to prevent stock shortages

## User Stories

- As a blood bank administrator, I want to add new blood units with ABO/Rh and collection date so that the inventory is up-to-date.
- As a nurse, I want to submit a transfusion request specifying blood type, Rh, and quantity so that I can quickly receive compatible units.
- As a blood bank technician, I want the system to automatically match requests to the most suitable units (exact-type first, then closest expiry) to optimize usage.
- As a manager, I want a dashboard showing current stock per type, units expiring within 7 days, and pending requests so I can make informed decisions.
- As a safety officer, I want the system to raise a shortage alert when any blood type falls below 5 units to ensure preparedness.

## Competitive Analysis

- BloodBankMS: Web-based, supports basic tracking but lacks automatic reservation release after 24h.
- Hemovigilance Pro: Comprehensive with integration to hospital EMR, but expensive and overcomplicated for small facilities.
- BloodTrack-IT: Cloud solution with mobile app; dashboard is strong but shortage alerts are email-only (no system notification).
- HemoSoft: Open-source, limited Rh factor handling and no expiry auto-expire feature.
- LifeStream Inventory: Focused on donor management rather than hospital inventory; compatibility logic minimal.
- Transfusion Safe: Good UI, but matching algorithm does not prioritize closest expiry; wastage higher.
- Blood Bank Manager by MedTech: Legacy desktop app, no real-time dashboard, requires manual alerts.

## Requirement Analysis

The system must track blood units with ABO type, Rh factor, and collection date; expiry is fixed at 42 days. It must accept transfusion requests (type, Rh, quantity) and match requests to compatible units using standard ABO/Rh donor compatibility, prioritizing exact-type matches first, then units closest to expiry. Reserved units that are not confirmed as issued within 24 hours must be released back to stock. Units past expiry must be automatically expired and excluded from matching. A shortage alert must be raised when available stock of any blood type falls below 5 units. The dashboard should display stock per type, units expiring within 7 days, and open requests. The system must handle concurrency (e.g., multiple requests simultaneously) and ensure data consistency. No user authentication or role-based access is specified, but it is implied for a real hospital environment. Integration with external systems (e.g., HL7) is not required initially.

## UI Design draft

Dashboard-centric layout with a top navigation bar (Dashboard, Inventory, Requests, Alerts). The Dashboard page displays three cards: Stock Overview (table with ABO type, Rh, available count, shortage indicator), Expiring Soon (table of units with expiry dates within 7 days), and Open Requests (list of pending requests with status). Inventory page allows adding, editing, and viewing blood units in a sortable grid. Requests page shows all requests with ability to fulfill manually or view automatic matches. Alerts page lists shortage notifications with timestamps. Style: clean, medical-themed (blue/white), responsive for desktop and tablet. Use simple icons and color-coded priority (red for shortage, yellow for expiring).

## Anything UNCLEAR

The requirement does not specify whether user authentication or role-based access is needed. For a realistic deployment, multiple user roles (admin, technician, nurse) would be necessary. Also, the matching algorithm details for Rh factor (e.g., Rh-negative can receive Rh-negative or Rh-positive? Standard compatibility: Rh-negative can receive Rh-negative only, Rh-positive can receive both) need to be clarified. The requirement says 'standard ABO/Rh donor compatibility' – we assume standard transfusion rules (e.g., O- is universal donor, AB+ universal recipient). Additionally, the alert mechanism is not detailed (dashboard only or email/SMS). The 24-hour reservation release should be implemented as a scheduled task or background job.
